# TRPG 地图管理工具 — 开发计划

## 一、背景与目标

面向 GM 的 TRPG 世界整理工具（**不是绘图工具**）。第一版做成 GitHub 上的本地小项目，方便 clone、`npm install`、`npm run dev` 跑起来内测，**暂不部署网站**。

核心体验：一张黄色羊皮纸风格的无限画布，GM 从左侧素材库拖拽黑白手绘风素材到画布，素材变成可编辑节点，用来整理地点、地形、聚落、遗迹、事件关系和自由备注。

**技术栈：** Vite + React + TypeScript + react-konva（画布）+ Zustand（状态）+ IndexedDB（本地存储）。

**已确认的三个关键决策：**

1. **内置素材**：第一版用代码生成的占位黑白线条 SVG（山/河/海/湖/森林/道路/村庄/城堡/遗迹/港口/标记 等），保证功能跑通，美术后续替换。
2. **存档格式**：导入/导出为单个 JSON 文件，用户素材图片以 base64 内嵌，导入即还原。
3. **范围**：做全部需求功能，但分阶段实现，每个阶段都能独立运行。

## 二、技术架构

```
src/
  main.tsx / App.tsx          # 路由：首页 <-> 编辑器
  routes/
    HomePage.tsx              # 世界管理首页
    EditorPage.tsx            # 编辑器（左素材库 / 中画布 / 右属性面板 / 顶工具栏）
  store/
    worldStore.ts             # 当前世界的画布数据（节点/连线/文本/视图）Zustand
    libraryStore.ts           # 素材库（内置 + 用户上传）
  db/
    idb.ts                    # IndexedDB 封装（基于 idb 库）
    schema.ts                 # 数据类型定义 + DB 版本
  canvas/                     # Konva 画布相关组件
    Canvas.tsx, NodeShape.tsx, EdgeLine.tsx, TextBox.tsx, ...
  panels/                     # 素材库面板、属性面板、工具栏
  assets/builtin/             # 内置占位 SVG
  serialization/
    worldFile.ts              # 导入/导出单 JSON（含 base64 图片）
  types.ts                    # 全局领域模型
```

### 数据模型（`src/types.ts` / `db/schema.ts`）

- `World`：`{ id, name, createdAt, updatedAt }` —— 元信息，存 IndexedDB `worlds` 表
- `WorldData`：`{ nodes, edges, texts, viewport }` —— 画布内容，按 worldId 存 `worldData` 表
- `MapNode`：`{ id, assetId, x, y, width, height, rotation, zIndex, name, description, customProps }`
- `CustomProp`：`{ id, label, type: 'text'|'checkbox'|'select'|'number'|'rating'|'progress'|'textarea', value, options? }`
- `Edge`：`{ id, fromNodeId, toNodeId, name, edgeType, description }`
- `TextBox`：`{ id, content, x, y, width, height, background, fontSize }`
- `Asset`：`{ id, name, builtin, kind, dataUrl }` —— 内置/用户素材，存 `assets` 表

**IndexedDB 表（用 `idb` 库）：** `worlds`、`worldData`、`assets`。
页面刷新数据不丢失 = 所有变更通过 store 落库（防抖保存）。

## 三、分阶段实现

### 阶段 0：脚手架（先能跑空壳）
- Vite + React + TS 模板；安装 `konva react-konva zustand idb react-router-dom`。
- `package.json` 脚本：`dev` / `build` / `preview`。
- 路由：首页 ↔ 编辑器。
- `README.md`：clone / install / dev / build 说明。
- **验收：** `npm install && npm run dev` 打开空首页；`npm run build` 通过。

### 阶段 1：世界管理首页 + IndexedDB
- `db/idb.ts` 封装：建库、`worlds`/`worldData`/`assets` 表 CRUD。
- 首页：创建世界、打开本地世界、删除世界、显示名称/创建时间/更新时间。
- 导入世界文件（JSON）入口（实现见阶段 6，先留按钮）。
- **验收：** 创建多个世界、刷新后仍在、能删除、能进入编辑器。

### 阶段 2：画布基础（羊皮纸 + 缩放平移）
- `canvas/Canvas.tsx`：Konva `Stage`/`Layer`，黄色羊皮纸背景。
- 无限画布：滚轮缩放（以鼠标为中心）、拖拽画布平移；viewport 存入 worldStore 并落库。
- 初始无任何预设内容。
- **验收：** 空白羊皮纸画布可自由缩放平移，刷新后视图位置保留。

### 阶段 3：素材库 + 拖拽生成节点
- `assets/builtin/`：生成约 11 类占位黑白线条 SVG。
- `libraryStore`：加载内置素材；左侧素材库面板按类别展示缩略图。
- 拖拽素材到画布 → 落点创建 `MapNode`（`Konva.Image` 渲染 SVG）。
- 节点操作：选中、移动、缩放（Transformer）、删除、复制。
- 层级调整：上移/下移/置顶/置底（改 `zIndex` 重排）。
- **验收：** 拖素材入画布成节点；可移动/缩放/删除/复制/调层级；刷新保留。

### 阶段 4：属性面板 + 自定义属性
- 右侧属性面板：选中节点显示名称、描述（默认仅这两项）。
- 自定义属性：用户自行添加属性行，控件类型支持 文本框 / 复选框 / 下拉框 / 数字 / 星级或进度格 / 多行文本。
- **不预设** 类型/阵营/危险等级/重要度等字段。
- **验收：** 给节点加多种自定义属性并编辑，数据持久化。

### 阶段 5：文本备注 + 连线
- 文本备注（`TextBox`）：顶部"文本"工具在任意位置新建，类似 Word 文本框；支持内容、位置、大小、背景样式；独立于节点，只是自由批注。
- 连线（`Edge`）：顶部"连线"工具，点两个节点建立连线；连线带名称、类型、描述（属性面板编辑）；节点移动时连线跟随。
- **验收：** 放置/编辑文本框；节点间连线并编辑名称/类型/描述；均持久化。

### 阶段 6：用户素材上传 + 导入导出
- 上传素材：支持 PNG / JPG / WEBP / SVG，转 dataUrl 存 `assets` 表；用户素材可删除、重命名；内置素材不可删除。
- 顶部"导出"：当前世界（元信息 + WorldData + 用到的用户素材 base64）序列化为单个 JSON 下载。
- 首页"导入世界文件"：读 JSON 还原成新世界（含素材）。
- **验收：** 上传/重命名/删除用户素材；导出 JSON 后在别处导入，节点/连线/文本/图片完整还原。

### 阶段 7：顶部工具栏整合 + 收尾
- 工具栏：选择 / 拖拽画布 / 连线 / 文本 / 导出（互斥的当前模式）。
- 打磨：快捷键（Delete 删除、Ctrl+C/V 复制）、空状态提示、基础错误处理。
- README 截图/使用说明补全。

## 四、关键复用与注意点

- **持久化统一走 store → 防抖写 IndexedDB**，避免每次拖动都写库。
- **react-konva 性能**：节点图片用 `Konva.Image`，缓存解码后的 image 对象；大量节点时单独 Layer。
- **素材渲染**：内置 SVG 与用户上传图片统一用 dataUrl → `HTMLImageElement` 加载，路径一致。
- **坐标系**：节点/文本/连线坐标统一存"画布世界坐标"，缩放平移只改 Stage 的 scale/position，不改数据坐标。
- **不过度设计**：第一版不做协作、不做撤销树（可留简单 undo 备选），优先保证能跑。

## 五、端到端验证

1. `npm install` → `npm run dev`，浏览器打开首页。
2. 创建世界 A → 进入编辑器 → 拖 3 个素材成节点 → 加自定义属性 → 连线 → 放文本框 → 刷新页面，数据完整保留。
3. 上传一张自定义 PNG 素材 → 拖入画布。
4. 导出世界 A 为 JSON → 回首页删除 A → 导入刚才的 JSON → 验证节点/连线/文本/图片完全还原。
5. `npm run build` 通过，`npm run preview` 可正常打开。
