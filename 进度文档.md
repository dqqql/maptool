## 二、设计基调（贯穿全程，勿偏离）

主题：**「羊皮纸地图志 / Aged Cartographer's Manuscript」**

- **配色**（`src/styles/global.css` 的 CSS 变量）：泛黄羊皮纸 `--paper-*`、棕黑墨水 `--ink-*`、火漆红点缀 `--seal`、暗金 `--gilt`。
- **字体**：标题毛笔楷书 `Ma Shan Zheng`（回退 STKaiti/KaiTi）；正文衬线 `Noto Serif SC`；拉丁古书体 `IM Fell English` / `Cinzel`。**严禁** Inter/Roboto/系统无衬线等「AI 味」字体。
- **纹理**：`.parchment-bg` 多层渐变模拟纸张纤维与暗角；`.grain-overlay` SVG `feTurbulence` 噪点（multiply 混合）。
- **手绘描边**：全局 `<svg>` 滤镜 `#sketch` / `#sketch-strong`（`feTurbulence + feDisplacementMap` 制造手抖位移），用 `.sketch-border` 或 `filter: url(#sketch)` 应用。
- **动效**：墨迹渗入 `ink-rise`（位移 + 模糊淡入）、`ink-in`、火漆盖印 `seal-pop`；卡片入场用 `animation-delay` 错落。统一缓动 `--ease`。已处理 `prefers-reduced-motion`。
- **交互文案**：契合地图志世界观（"开辟新世界""焚毁此卷""回到原点"），而非通用 UI 措辞。

> 复用约定：新增 UI 一律取用上述 CSS 变量与工具类（`.ink-btn` / `.ink-field` / `.sketch-border` / `.rail` 等），不要硬编码颜色，保证整体氛围一致。

## 三、各阶段落地细节

### 阶段 0 · 脚手架 ✅
- `package.json`：`dev` / `build`（`tsc -b && vite build`）/ `preview`。
- 依赖：`react` `react-dom` `react-router-dom` `konva` `react-konva` `zustand` `idb`。
- `vite.config.ts`（端口 5173，自动打开）；`tsconfig.json` + `tsconfig.node.json`。
- 路由用 `HashRouter`（本地直开 `dist/index.html` 也能跑）：`/` 首页 ↔ `/world/:id` 编辑器。
- `README.md`、`.gitignore` 就绪。
- **验收**：`npm install && npm run dev` 打开首页；`npm run build` 通过 ✅。

### 阶段 1 · 世界管理首页 + IndexedDB ✅
- `src/db/schema.ts`：库名 `trpg-maptool` v1，三表 `worlds`（`by-updatedAt` 索引）/ `worldData` / `assets`。
- `src/db/idb.ts`：`listWorlds` / `createWorld`（同时建空 worldData）/ `deleteWorld`（级联删 worldData）/ `getWorld` / `touchWorld` / `getWorldData` / `putWorldData`，以及 assets 的 CRUD（为阶段 6 预备）。
- `src/routes/HomePage.tsx`：报头（罗盘玫瑰 + 毛笔标题 + 手绘分隔线）、操作条、世界卡片网格（折角纸片 + 悬浮微旋 + 悬浮删除键）、空状态卷轴插画、创建弹层、删除确认弹层。
- 「导入舆图」按钮已放置但禁用（阶段 6 启用）。
- **验收**：创建多个世界、刷新仍在、可删除、可进入编辑器 ✅。

### 阶段 2 · 画布基础 ✅
- `src/store/worldStore.ts`：Zustand 管理当前世界 `viewport`，`load/unload/setViewport`，**500ms 防抖写库**（`putWorldData` + `touchWorld`），避免拖动时频繁落库。
- `src/canvas/Canvas.tsx`：`react-konva` `Stage/Layer`（透明叠加，纸张纹理由 CSS 提供）。
  - 网格在「世界坐标」绘制并随视口缩放平移，主/次线 + 原点十字标记，`strokeWidth` 按 `1/scale` 保持视觉恒定。
  - 滚轮缩放以鼠标为锚点；空白处拖拽平移（命中节点不平移，为阶段 3 预留）；缩放区间 0.2–4。
  - `ResizeObserver` 自适应容器尺寸。
- `src/routes/EditorPage.tsx`：顶栏（返回 / 世界名 / 工具按钮占位）+ 三栏（左素材库占位、中画布、右属性占位）+ 画布 HUD（缩放 ± / 百分比重置 / 回到原点）。
- 坐标系遵循约定：数据坐标不变，仅改 Stage 的 `scale/position`。
- **验收**：空白羊皮纸可自由缩放平移，刷新后视图位置保留 ✅。

### 阶段 3 · 素材库 + 拖拽生成节点 ✅
- `src/assets/builtin.ts`：代码生成 **11 类**黑白手绘线条 SVG（山岳/丘陵/森林/河流/湖泊/海洋/道路/村庄/城堡/遗迹/港口/标记），统一墨色描边、圆头线帽；按「地形/水文/聚落/人文/标记」分组；`dataUrl` 内嵌，美术可后续替换（保持 `kind`）。
- `src/store/libraryStore.ts`：内置素材（不入库）+ 用户素材（阶段 6 从 assets 表加载）；`getAsset(id)` 供节点渲染查素材。
- `src/panels/LibraryPanel.tsx`：左栏分组缩略图，HTML5 `draggable`，**拖入画布**或**双击落于视图中心**生成节点。
- `src/canvas/NodeShape.tsx` + `useImage.ts`：`Konva.Image` 渲染素材（带缓存的 `HTMLImageElement`），选中外框 + 名称标签；拖拽移动、`Transformer` 缩放/旋转回写。
- `src/canvas/Canvas.tsx`：节点层 + 单一 `Transformer` 绑定选中节点；空白处点击取消选中并平移；`onDrop` 屏幕坐标→世界坐标生成节点；节点按 `zIndex` 排序渲染。
- worldStore 扩展：`addNode/updateNode/removeNode/duplicateNode/select` 与层级 `bringToFront/sendToBack/bringForward/sendBackward`；删除节点级联删相关连线。
- **验收**：拖素材入画成节点；可移动/缩放/删除/复制/调层级；刷新保留 ✅。

### 阶段 4 · 属性面板 + 自定义属性 ✅
- `src/panels/PropertyPanel.tsx`：右栏选中节点显示**名称**、**描述**（默认仅此两项，未预设阵营/危险度等字段，符合计划要求）。
- 节点操作条：置顶/上移/下移/置底、复制、删除。
- 自定义属性：用户自行添加属性行，控件类型 7 种 —— 文本框 / 多行文本 / 复选框 / 下拉框（可填选项）/ 数字 / 星级 / 进度格。
- `src/panels/CustomPropRow.tsx`：星级（★ 可点选、再点同星归零）、进度格（斜纹填充段）、复选框等均为手绘风自定义控件。
- 全部经 store 落库（500ms 防抖），刷新持久化。
- 快捷键：`Delete/Backspace` 删除选中、`Ctrl/⌘+D` 复制（输入框内不触发；完整快捷键体系仍归阶段 7）。
- **验收**：给节点加多种自定义属性并编辑，数据持久化 ✅。

### 阶段 5 · 文本备注 + 连线 ✅
- 连线 `src/canvas/EdgeLine.tsx`：手绘弧线（轻微垂直起拱）+ 端点 + 中点名称标签；**随节点移动跟随**（坐标实时取自节点中心）；透明粗线作命中区便于点选。
- 文本框 `src/canvas/TextShape.tsx`：背景纸片 + 文字，可拖拽/缩放；**双击进入 HTML `<textarea>` 浮层编辑**（屏幕坐标随视口换算）；空内容显示占位提示。
- 连线模式：依次点选两个节点建立连线（起点高亮环、去重、禁止自连）；文本模式：空白处点击新建文本框。
- 属性面板按选中类型切换：连线编辑名称/类型/描述（显示 起点⟿终点）；文本编辑内容/字号滑块/背景色板。
- **验收**：放置/编辑文本框；节点间连线并编辑名称/类型/描述；均持久化、刷新保留 ✅。

### 阶段 6 · 用户素材上传 + 导入导出 ✅
- 上传：`libraryStore.uploadFiles` 接收 PNG/JPG/WEBP/SVG → `FileReader` 转 dataUrl 存 `assets` 表；左栏「我的素材」可重命名（双击/✎）、删除（✕）；内置素材不可删。
- 导出：`src/serialization/worldFile.ts` `exportWorld` 将 世界元信息 + WorldData + **节点用到的用户素材（base64 内嵌）** 序列化为单个 `*.world.json` 下载；内置素材不导出（靠 id 还原）。
- 导入：首页「导入舆图」→ `importWorldFromFile` 校验格式、还原用户素材（保留原 id 使节点引用有效）、新建世界并写入画布，随即进入编辑器。
- **验收**：上传/重命名/删除用户素材；导出 JSON 后导入，节点/连线/文本/图片完整还原（端到端实测通过）✅。

### 阶段 7 · 工具栏整合 + 收尾 ✅
- 顶栏互斥模式工具：选择 / 漫游(平移) / 连线 / 文本，外加导出动作；`aria-pressed` 高亮当前模式，连线/文本模式显示顶部提示横幅。
- 快捷键：`Delete/Backspace` 删除选中（节点/连线/文本）、`Ctrl/⌘+C` 复制 + `Ctrl/⌘+V` 粘贴、`Ctrl/⌘+D` 直接复制、`Esc` 退出当前模式并取消选中（输入框内不触发）。
- 空状态提示、画布底部缩放 HUD、错误处理（导入/导出失败 alert）。
- README 使用说明补全。
- **验收**：端到端流程（创建→放素材→连线→文本→上传→导出→导入还原）经 Playwright 跑通，零控制台错误 ✅。
